# Документация по коду

Этот документ описывает архитектуру и устройство кода. Для инструкций
по запуску и деплою смотрите [README.md](README.md).

## Архитектура

```
┌────────────────┐   HTTP/JSON    ┌────────────────────────┐
│  React (Vite)  │ ─────────────▶ │  Express API           │
│  client/       │                │  server/               │
│                │ ◀───────────── │  └─ JSON file storage  │
└────────────────┘                └────────────────────────┘
```

- В **dev** Vite (порт `5173`) проксирует `/api/*` на Express (`4000`).
- В **production** Express отдаёт собранный `client/dist` и тот же
  REST API. Один процесс, один порт.

## Модель данных

Каталог — **рекурсивное дерево узлов**. Узел может быть либо группой
(содержит дочерние узлы), либо листом-работой (содержит цену):

```ts
type Node = {
  id: string;          // UUID
  title: string;       // название (обязательно)
  measure?: string;    // ед. изм. — наследуется потомками
  price?: number;      // ЛИСТ — фиксированная цена
  children?: Node[];   // ГРУППА — вложенные узлы
};
```

Правила:

- Узел **не может** одновременно иметь и `price`, и `children` —
  это нарушение схемы и будет отклонено сервером.
- Глубина дерева не ограничена. Любой лист (узел с ценой) на любом
  уровне — выбираемая работа.
- `measure` берётся с самого узла, иначе с ближайшего предка.
  Это позволяет указать единицу измерения один раз на уровне группы.

Примеры:

```jsonc
// Группа с подгруппами и листами:
{ "title": "Монтаж кабеля", "measure": "м.", "children": [
  { "title": "3-х жильный", "children": [
    { "title": "Сечение 1,5мм²", "price": 70 },
    { "title": "Сечение 2,5мм²", "price": 70 }
  ]}
]}

// Простой лист — работа без вариантов:
{ "title": "Монтаж теплого пола", "measure": "м²", "price": 700 }
```

## Серверная часть (`server/`)

| Файл                       | Назначение                                                |
| -------------------------- | --------------------------------------------------------- |
| `src/index.js`             | Express bootstrap, статика, health-check                  |
| `src/routes/works.js`      | REST: `GET/POST/PUT/DELETE /api/works`                    |
| `src/storage.js`           | Чтение/атомарная запись JSON-файла, кэш в памяти          |
| `data/works.seed.json`     | Стартовые данные (используются при первом запуске)        |
| `data/works.json`          | Рабочее хранилище (создаётся автоматически)               |

### REST API

| Метод   | Путь              | Тело                                   | Ответ          |
| ------- | ----------------- | -------------------------------------- | -------------- |
| GET     | `/api/health`     | —                                      | `{ ok: true }` |
| GET     | `/api/works`      | —                                      | `Node[]`       |
| POST    | `/api/works`      | `Node & { parentId?: string }`         | `Node`         |
| PUT     | `/api/works/:id`  | `Node`                                 | `Node`         |
| DELETE  | `/api/works/:id`  | —                                      | `204`          |
| PUT     | `/api/works`      | `Node[]` (полная замена дерева)        | `Node[]`       |

`POST /api/works` без `parentId` добавляет узел в корень. С `parentId` —
в `children` указанного узла (узел должен быть группой).
Удаление по `id` рекурсивно ищет узел в дереве.

Редактор каталога в UI использует `PUT /api/works` (массовая замена) —
все правки применяются одной транзакцией.

Запись на диск идёт через `tmp + rename`, что исключает повреждённый
файл при падении процесса.

### Адаптивность

Лейаут построен на CSS Grid:

```css
.layout { grid-template-columns: 1fr; }                  /* мобильные */
@media (min-width: 900px) {
  .layout { grid-template-columns: minmax(360px, 1fr) 1.4fr; }
}
```

### Темизация

Переменные CSS объявлены в `:root` и переопределяются для
`[data-theme="dark"]`. Компонент `ThemeToggle` пишет атрибут
`data-theme` на `<html>` и сохраняет выбор в `localStorage`.

### Состояние

- **Каталог** загружается один раз через `api.listWorks()` и
  перечитывается после изменений в редакторе.
- **Список позиций** — локальный стейт `App` (не сохраняется на сервере;
  предназначен для одной сессии и экспортируется в Excel).
- **Тема** — `localStorage` ключ `theme`.

## CI/CD

`.github/workflows/deploy.yml`:

- **build** — на каждый push: `npm install`, `npm run build`,
  загрузка артефакта `client-dist`.
- **deploy** — только для `main`: подключение по SSH и запуск
  `scripts/deploy.sh` на сервере. Перед обновлением деплой-скрипт
  делает бэкап.

## Безопасность

- API не имеет встроенной авторизации — используйте VPN, базовую
  HTTP-авторизацию nginx или вынесите редактор каталога в защищённую
  сеть. Чтение каталога безопасно (read-only).
- Лимит JSON-тела установлен в 1 MB (`express.json({ limit: "1mb" })`).
- Атомарная запись хранилища предотвращает повреждение файла.

## Расширение

- **Сохранение смет на сервере** — добавьте роут `/api/estimates`
  с моделью `Estimate { id, createdAt, items[] }` по образцу `works.js`.
- **БД вместо JSON** — замените `storage.js` (например, на SQLite через
  `better-sqlite3`), API не изменится.
- **Авторизация** — добавьте middleware (например, JWT) перед `worksRouter`.
